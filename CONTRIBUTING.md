# Contributing

Thanks for helping improve the Base64 Cheatsheet. Small fixes and new examples are both welcome.

## Ways to contribute

- Fix a typo, broken link, or unclear sentence.
- Add a real-world example in an existing language.
- Add a new language or tool (Go, Rust, Ruby, PHP, Java, .NET, etc.).
- Improve an explanation or table.
- Suggest a better structure or missing topic.

## Before you start

- Keep examples short and self-contained.
- Use real input/output in code blocks so readers can verify them.
- Prefer the standard library in any language. Only reach for a third-party package if it's clearly the idiomatic choice.
- Avoid marketing language. This is a reference, not a landing page.

## Workflow

1. Fork the repository.
2. Create a branch: `git checkout -b my-improvement`.
3. Make your changes. Run your examples to confirm they work.
4. Commit with a clear message, e.g. `Add Go base64 example`.
5. Open a pull request describing the change and why it's useful.

## Style

- Clean Markdown, no HTML unless necessary.
- Fenced code blocks with a language hint (` ```js `, ` ```python `, ` ```sh `).
- Tables for comparisons.
- Short paragraphs. Favor clarity over cleverness.

## Reporting issues

If something is wrong or missing, open an issue with:

- What you expected
- What you saw
- A minimal example if relevant

No issue is too small. A one-line typo fix is just as welcome as a new section.
